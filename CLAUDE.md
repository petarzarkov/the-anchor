# CLAUDE.md

Guidance for Claude when working in this repository.

## Architecture

- `app/` — React 19 frontend (TypeScript, Chakra UI v3, Vite)
- `service/` — Rust backend (Tauri 2 commands and shared state)
- `service/src/main.rs` — the only Rust source file; contains `AppState`, all `#[tauri::command]` functions, and unit tests
- `app/theme/` — Chakra UI customization: `system.ts` (tokens, keyframes) and `recipes.ts` (button/dialog variants)

## Package Manager and Scripts

Always use **Bun**, never npm or yarn.

```bash
bun install
bun run dev           # frontend only (Vite, port 1420)
bun run tauri:dev     # full app (Rust + frontend)
bun run check         # biome ci — CI-safe, no writes
bun run lint          # biome check --write --unsafe — fixes in place
bun run typecheck     # tsc --noEmit
bun run build         # tsc + vite build
```

For Rust:
```bash
cargo test --manifest-path service/Cargo.toml
```

## Code Style

Biome enforces formatting. Key settings from `biome.json`:
- **Line width: 60 characters** — JSX props go on separate lines quickly
- Single quotes, trailing commas, arrow parens only when needed
- `noExplicitAny` is an error
- Max 500 lines per file, max cognitive complexity 20

When writing or editing TypeScript/TSX, pre-format to 60-char width so `bun run check` passes without needing a fix run.

## Chakra UI Custom Variants

`buttonRecipe` in `app/theme/recipes.ts` defines: `solid`, `ghost`, `outline`, `fire`, `glass`.

`fire` and `glass` are **not in Chakra's default prop types** — using `variant="fire"` causes a TS error. Workaround: `variant={"fire" as "solid"}`. The proper fix is running `bunx @chakra-ui/cli typegen` but that is not yet set up. Avoid using `fire`/`glass` in demo code to keep zero type errors.

## Rust Conventions

- All commands are **synchronous** — no `async fn` needed for simple state reads/writes
- `AppState` uses `std::sync::Mutex<i32>` — lock, mutate, return the new value
- Return primitives (`i32`, `String`, `bool`) directly; Tauri serializes them via its bundled serde
- `serde_json` **must** be a direct dep — `tauri::generate_context!()` references it by name in its macro expansion
- Do not add `serde`, `tokio`, or `log` as direct deps unless they are explicitly called — Tauri provides them transitively

## Adding a New Tauri Command

1. Write the function in `service/src/main.rs` with `#[tauri::command]`
2. Add it to `tauri::generate_handler![...]` in `main()`
3. If it needs state, use `state: State<'_, AppState>` — no other setup required
4. Call it from React: `await invoke<ReturnType>('command_name')`
5. Write a unit test in the `#[cfg(test)]` block — test the logic directly on `AppState`, not through Tauri's runtime

## CI

Two parallel jobs in `.github/workflows/ci.yml`:
- **frontend** — skips Rust (`setup-rust: 'false'`), runs `bun run --parallel check typecheck` then `bun run build`
- **rust** — installs Linux system deps (webkit2gtk etc.), runs `cargo test`

The reusable setup action is at `.github/actions/setup/action.yml`. It has two relevant inputs: `setup-rust` (default `'true'`) and `rust-targets` (default `''` = native).

## Generated Files

`service/gen/` is **gitignored and biome-ignored** — it is auto-generated by Tauri's build script. Never edit files there manually.

## Known Gotchas

- `service/Cargo.lock` is untracked — this is intentional for a template (consumers manage their own lockfile)
- The `mock` feature in `Cargo.toml` (`default = ["mock"]`) is a placeholder — nothing checks `#[cfg(feature = "mock")]` yet
- Husky hooks must be executable (`100755`) — use `git update-index --chmod=+x .husky/<hook>` if they lose the bit
- `bun run lint` uses `biome check --write --unsafe` which lints the whole project, ignoring lint-staged's file arguments — this is acceptable for a small template
